项目概述

这是一个基于Bootstrap 5的现代化Web应用开发框架，旨在帮助开发人员快速构建响应式、移动优先的网站和Web应用程序。

主要特性：
  • 完全响应式设计，适配所有设备尺寸
  • 基于Bootstrap 5的现代化UI组件
  • 模块化结构，易于定制和扩展
  • 详细的文档和示例代码
  • 内置主题系统，支持深色/浅色模式
  • 优化的性能和加载速度
当前开发进度 75% 完成

快速开始

安装指南
通过npm安装

推荐使用npm进行安装，以便管理依赖关系：

$ npm install my-project --save
通过CDN引入

直接在HTML文件中引入CDN链接：

<!-- CSS -->
<link href="https://cdn.example.com/my-project.min.css" rel="stylesheet">

<!-- JavaScript -->
<script src="https://cdn.example.com/my-project.min.js"></script>
基本模板

使用以下HTML模板开始您的项目：

<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>我的项目</title>
  <link href="my-project.min.css" rel="stylesheet">
</head>
<body>
  <!-- 页面内容 -->
  <script src="my-project.min.js"></script>
</body>
</html>

配置指南

本部分包含系统所有可配置选项的详细说明。配置文件通常位于项目根目录下的 config/ 文件夹中。
基础配置

config/app.js

配置项 类型 默认值 描述
name string 'MyApp' 应用程序名称
env string 'production' 应用环境 (production, development, testing)
debug boolean false 是否启用调试模式
url string 'http://localhost' 应用程序URL
timezone string 'UTC' 默认时区
locale string 'en' 默认语言
配置示例
module.exports = {
  name: 'My Application',
  env: process.env.NODE_ENV || 'development',
  debug: process.env.NODE_ENV === 'development',
  url: process.env.APP_URL || 'http://localhost:3000',
  timezone: 'Asia/Shanghai',
  locale: 'zh-CN'
};
数据库配置

config/database.js

配置项 类型 默认值 描述
default string 'mysql' 默认数据库连接
connections.mysql.host string '127.0.0.1' MySQL服务器地址
connections.mysql.port number 3306 MySQL服务器端口
connections.mysql.database string 'forge' 数据库名称
connections.mysql.username string 'forge' 数据库用户名
connections.mysql.password string '' 数据库密码
connections.mysql.charset string 'utf8mb4' 数据库字符集
connections.sqlite.database string 'storage/database.sqlite' SQLite数据库文件路径
配置示例
module.exports = {
  default: 'mysql',

  connections: {
    mysql: {
      host: process.env.DB_HOST || '127.0.0.1',
      port: process.env.DB_PORT || 3306,
      database: process.env.DB_DATABASE || 'myapp',
      username: process.env.DB_USERNAME || 'root',
      password: process.env.DB_PASSWORD || '',
      charset: 'utf8mb4',
      timezone: '+08:00'
    },

    sqlite: {
      database: 'storage/database.sqlite'
    }
  }
};
安全提示： 生产环境中请勿在配置文件中直接写入敏感信息，应使用环境变量。
缓存配置

config/cache.js

配置项 类型 默认值 描述
default string 'file' 默认缓存驱动
stores.file.location string 'storage/cache' 文件缓存存储路径
stores.redis.connection string 'default' Redis连接名称
prefix string 'myapp_cache' 缓存键前缀
ttl number 3600 默认缓存时间(秒)
支持的缓存驱动
  • file 默认
  • redis
  • memcached
  • database
  • array
  • null (禁用缓存)
邮件配置

config/mail.js

配置项 类型 默认值 描述
default string 'smtp' 默认邮件驱动
mailers.smtp.host string 'smtp.mailgun.org' SMTP服务器地址
mailers.smtp.port number 587 SMTP服务器端口
mailers.smtp.username string null SMTP用户名
mailers.smtp.password string null SMTP密码
mailers.smtp.encryption string 'tls' 加密方式 (tls, ssl)
from.address string 'hello@example.com' 默认发件人邮箱
from.name string 'Example' 默认发件人名称
常用邮件服务配置

mailers: {
  smtp: {
    host: 'smtp.gmail.com',
    port: 587,
    username: 'your@gmail.com',
    password: 'your-app-password', // 使用应用专用密码
    encryption: 'tls'
  }
}
使用Gmail需要启用"安全性较低的应用"或创建应用专用密码。

mailers: {
  mailgun: {
    domain: 'your-domain.com',
    secret: 'your-mailgun-key',
    endpoint: 'api.mailgun.net' // 欧盟区域使用 'api.eu.mailgun.net'
  }
},
default: 'mailgun'
.env 环境变量

以下是最常用的环境变量配置，应存储在项目根目录的 .env 文件中：

# 应用配置
APP_NAME=MyApp
APP_ENV=production
APP_KEY=base64:your-random-app-key
APP_DEBUG=false
APP_URL=http://localhost

# 数据库配置
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=myapp
DB_USERNAME=root
DB_PASSWORD=

# 邮件配置
MAIL_MAILER=smtp
MAIL_HOST=smtp.mailtrap.io
MAIL_PORT=2525
MAIL_USERNAME=null
MAIL_PASSWORD=null
MAIL_ENCRYPTION=tls
MAIL_FROM_ADDRESS=hello@example.com
MAIL_FROM_NAME="${APP_NAME}"

# Redis配置
REDIS_HOST=127.0.0.1
REDIS_PASSWORD=null
REDIS_PORT=6379

# 第三方服务
GOOGLE_CLIENT_ID=
GOOGLE_CLIENT_SECRET=
FACEBOOK_CLIENT_ID=
FACEBOOK_CLIENT_SECRET=
提示： 在开发环境中，可以复制 .env.example 文件并重命名为 .env，然后修改相应配置。

API 参考

API 基础 URL: http:///api/public/
所有API请求都需要在请求头中包含 Authorization: Bearer <your_token>
认证 API

POST /auth/login

需要验证

用户登录接口，成功登录后返回访问令牌。

请求参数
参数 类型 必填 描述
username string 是 用户名或邮箱
password string 是 用户密码
remember boolean 否 是否记住登录状态
请求示例
// cURL 示例
curl -X POST \
  https://api.example.com/v1/auth/login \
  -H 'Content-Type: application/json' \
  -d '{
    "username": "admin@example.com",
    "password": "yourpassword",
    "remember": true
  }'
响应示例
// 成功响应 (200 OK)
{
  "success": true,
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "expires_in": 3600,
    "user": {
      "id": 123,
      "name": "Admin User",
      "email": "admin@example.com",
      "roles": ["admin"]
    }
  }
}

// 错误响应 (401 Unauthorized)
{
  "success": false,
  "error": {
    "code": "AUTH_FAILED",
    "message": "用户名或密码错误"
  }
}

POST /auth/refresh

需要验证

使用刷新令牌获取新的访问令牌。

请求示例
curl -X POST \
  https://api.example.com/v1/auth/refresh \
  -H 'Authorization: Bearer <refresh_token>'
用户管理 API

GET /users

管理员权限

获取用户列表，支持分页和筛选。

查询参数
参数 类型 必填 描述
page integer 否 页码，默认为1
per_page integer 否 每页数量，默认为15
search string 否 搜索关键词（用户名或邮箱）
status string 否 用户状态（active, suspended, pending）
sort string 否 排序字段（id, name, created_at等）
order string 否 排序方向（asc, desc）
请求示例
curl -X GET \
  "https://api.example.com/v1/users?page=1&per_page=10&search=admin&status=active" \
  -H 'Authorization: Bearer <your_token>'
响应示例
{
  "success": true,
  "data": {
    "current_page": 1,
    "data": [
      {
        "id": 1,
        "name": "Admin User",
        "email": "admin@example.com",
        "status": "active",
        "created_at": "2023-01-15T08:30:00Z"
      },
      // 更多用户数据...
    ],
    "total": 42,
    "per_page": 10,
    "last_page": 5
  }
}

POST /users

管理员权限

创建新用户账户。

请求参数
参数 类型 必填 描述
name string 是 用户全名
email string 是 用户邮箱
password string 是 用户密码（至少8个字符）
roles array 否 用户角色数组
status string 否 用户状态（默认为active）
请求示例
curl -X POST \
  https://api.example.com/v1/users \
  -H 'Authorization: Bearer <your_token>' \
  -H 'Content-Type: application/json' \
  -d '{
    "name": "New User",
    "email": "new.user@example.com",
    "password": "securepassword123",
    "roles": ["editor"],
    "status": "active"
  }'

GET /users/{id}

需要验证

获取单个用户的详细信息。

路径参数
参数 类型 描述
id integer 用户ID

PUT /users/{id}

需要验证

更新用户信息。

普通用户只能更新自己的信息，管理员可以更新所有用户信息。
错误代码

以下是API可能返回的错误代码及其含义：

HTTP 状态码 错误代码 描述 可能原因
400 BAD_REQUEST 请求参数无效 缺少必要参数或参数格式错误
401 UNAUTHORIZED 未授权访问 未提供token或token无效
403 FORBIDDEN 禁止访问 权限不足
404 NOT_FOUND 资源不存在 请求的API端点或资源不存在
429 TOO_MANY_REQUESTS 请求过于频繁 超出API调用频率限制
500 INTERNAL_ERROR 服务器内部错误 服务器端处理请求时发生错误
错误响应格式
{
  "success": false,
  "error": {
    "code": "ERROR_CODE",
    "message": "错误描述信息",
    "details": {
      // 可选，包含额外的错误详情
    }
  }
}
